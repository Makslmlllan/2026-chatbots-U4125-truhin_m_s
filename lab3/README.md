# 1. Описание проекта

В рамках лабораторной работы был разработан Telegram-бот **SmartDiary**, предназначенный для управления дедлайнами и задачами.

Функциональность бота:

* добавление дедлайнов
* просмотр списка задач
* просмотр ближайших дедлайнов
* удаление задач

Цель лабораторной работы — развернуть бота в облачной среде и обеспечить его доступность для реальных пользователей.

# 2. Выбор способа деплоя

# 3. Подготовка к деплою

Перед деплоем были выполнены следующие шаги:

* Проверена работоспособность бота локально
* Токен бота вынесен в переменные окружения
* Создан файл `.gitignore`:

```
.env
__pycache__/
*.pyc
*.db
*.log
```

* Создан файл `requirements.txt` с зависимостями:

```
python-telegram-bot==20.7
python-dotenv==1.0.0
pandas==2.1.4
requests==2.31.0
```

* Добавлено логирование:

```python
import logging

logging.basicConfig(
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    level=logging.INFO
)
```

# 4. Процесс деплоя

Деплой осуществлялся через платформу Railway:

1. Репозиторий был загружен на GitHub
2. Проект подключен к Railway
3. Указана рабочая директория:

```
lab1
```

4. Добавлена переменная окружения:

```
BOT_TOKEN=***
```

5. Указана команда запуска:

```
python bot.py
```

После этого Railway автоматически:

* установил зависимости
* собрал проект
* запустил бота

# 5. Тестирование

После деплоя был проведён тест функциональности бота.

Проверены следующие сценарии:

### Основные команды

* `/start` — запуск бота
* `/add_deadline` — добавление задачи
* `/list` — просмотр списка задач
* `/nearest` — просмотр ближайших дедлайнов
* `/delete` — удаление задачи

### Проверка ошибок

* ввод некорректной даты
* ввод некорректного времени
* удаление несуществующего ID

### Проверка стабильности

* бот работал без локального запуска
* проверена работа спустя длительное время
